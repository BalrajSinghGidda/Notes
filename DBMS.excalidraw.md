External Schema ():

Each user sees only the data relevant to them.

It enhances security and simplicity.

It is independent of the physical storage. ^PtxojSVO

Keys and Constraints: ^yObCOtuj

A key is an attribute that helps to identify a record from the table,
and establish the relationships between them.

Types:

- SuperKeys:
   -- It is a superset, from which we derive all the keys.
   -- It uniquely identifies the tuple.
   -- It can contain NULL values.
   -- It can contain extra values/attributes.

- Candidate Key:
   -- The minimal superkey is called a candidate key.

- Primary Key:
   -- It is used to denote a candidate key.
   -- Primary key= Unique + Not NULL
   -- It is chosen by DBA.

- Alternate Key:
   -- The candidate key other than the primary key are alternate keys.

- Unique Key:
   -- 100% unique key
   -- Can be NULL.

- Composite Key:
   -- It contains 2 or more attributes.

- Foreign Key:
